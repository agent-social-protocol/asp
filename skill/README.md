# Public Skills

This repo keeps one canonical public protocol-level social skill:

- `social/SKILL.md` — CLI-first, MCP-compatible ASP social workflow guidance

App-facing skills should live in their own public repos under the product org,
not in the protocol repo. They should stay thin:
inherit the protocol workflow, add branded entry semantics or product-owned
extensions, and avoid becoming a second source of truth for core ASP behavior.

Older experimental skill variants were removed from the public surface because
they are no longer maintained.
